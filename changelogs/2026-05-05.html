WIF reference

New

New exhaustive reference for Workload Identity Federation covering the token exchange response schema, all SDK environment variables, credential precedence order, profile configuration file format, OAuth scopes, issuer/rule validation constraints, JWT verification rules, CEL rule-matching semantics, JWKS source modes, token exchange error codes, and a step-by-step troubleshooting guide.

  • Token exchange response — Documents POST /v1/oauth/token RFC 6749 response fields: access_token (sk-ant-oat01-... prefix), token_type, expires_in, and scope
  • Environment variables — Specifies all six SDK federation variables: ANTHROPIC_FEDERATION_RULE_ID, ANTHROPIC_ORGANIZATION_ID, ANTHROPIC_IDENTITY_TOKEN_FILE, ANTHROPIC_IDENTITY_TOKEN, ANTHROPIC_SERVICE_ACCOUNT_ID, and ANTHROPIC_PROFILE, including which combinations activate the federation path
  • Credential precedence — Defines the five-level resolution order: constructor argument → ANTHROPIC_API_KEY/ANTHROPIC_AUTH_TOKEN → ANTHROPIC_PROFILE → federation environment variables → active profile; warns that empty-string variables still win their precedence slot
  • Profile configuration — Documents the ~/.config/anthropic/configs/ JSON profile format, active profile resolution order, and the split between the non-secret config file and the secret credentials file (mode 0600)
  • OAuth scopes — Defines the workspace:developer scope as granting access to all non-administrative Claude API endpoints (Messages, Models, Managed Agents, Files, Skills)
  • Validation rules — Enumerates constraints on resource names, token_lifetime_seconds (60–86400), URL fields (HTTPS, port 443, public DNS), and JWT verification (16 KiB max, asymmetric algorithms only, 30-second clock skew)

  • Token exchange errors — Maps all 400 invalid_grant causes and common SDK-side failures to resolution steps; references the authentication history page in the Claude Console for live debugging

Automate workflows with hooks

Updated

Updated the description of what happens when an LLM hook returns "ok": false to clarify that behavior is now event-specific rather than uniform.

  • PreToolUse ok: false — Now explicitly documented: the tool call is denied and the reason is returned to Claude as the tool error so it can adjust and continue, rather than ending the turn
  • PostToolUse / PostToolBatch / UserPromptSubmit / UserPromptExpansion — Confirmed to end the turn with the reason appearing as a chat warning line; Claude does not see it
  • Stop / SubagentStop — Unchanged: reason is fed back to Claude as its next instruction so it keeps working

Workload Identity Federation

New

New primary WIF concept and setup guide explaining how service accounts, federation issuers, and federation rules work together, walking through the three-step Console setup, and documenting credential precedence, API key migration, token lifetime, and provider-specific links.

  • Core concepts — Defines service accounts (svac_...), federation issuers (fdis_...), and federation rules (fdrl_...) and explains how they compose: issuers establish trust, rules define JWT-to-service-account mappings with match conditions and token lifetime, service accounts are the billable principals
  • Token exchange flow — Documents the RFC 7523 jwt-bearer grant at POST /v1/oauth/token: SDK posts IdP JWT, Anthropic verifies signature against registered JWKS, checks exp/nbf/iat, evaluates rule match, returns sk-ant-oat01-... access token; SDK refreshes before expiry
  • Console setup walkthrough — Three-step guide: (1) register issuer with URL, JWKS source, and optional CA cert; (2) create service account and add to workspace; (3) create federation rule with match conditions, target service account, scope, and token lifetime
  • Credential precedence — Describes the five-level resolution order and the zero-argument client pattern for shipping the same container image across environments
  • Migration from API keys — Advises unsetting ANTHROPIC_API_KEY/ANTHROPIC_AUTH_TOKEN when migrating, since non-empty values shadow federation entirely
  • Provider guides — Index of per-provider setup guides for AWS, GCP, Azure, GitHub Actions, Kubernetes, and Okta
