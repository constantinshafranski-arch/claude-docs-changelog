Monitoring

Updated

Adds the effort attribute to cost counter, token counter, API request, and API error telemetry events, and introduces command_name and command_source fields on user_prompt events with OTEL_LOG_TOOL_DETAILS behaviour extended to cover custom/plugin/MCP command names.

  • effort attribute — New effort attribute (low, medium, high, xhigh, max) added to cost counter, token counter, API request event, and API error event; absent when the model does not support effort
  • command_name field — New command_name field on user_prompt events; built-in commands emit canonical names, custom/plugin/MCP commands collapse to custom or mcp unless OTEL_LOG_TOOL_DETAILS=1
  • command_source field — New command_source field on user_prompt events: builtin, custom, or mcp; plugin-provided commands report as custom
  • OTEL_LOG_TOOL_DETAILS extended — When enabled, also surfaces verbatim custom, plugin, and MCP command names on user_prompt events in addition to existing tool parameter logging
