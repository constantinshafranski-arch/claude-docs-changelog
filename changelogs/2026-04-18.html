Release Notes

Key Highlights

  • Claude in Amazon Bedrock GA — The Messages-API Bedrock endpoint (/anthropic/v1/messages) is now open to all Amazon Bedrock customers; Claude Opus 4.7 and Haiku 4.5 are available self-serve across 27 AWS regions, ending the invite-only research preview.
  • New Bedrock Mantle clients across all SDKs — Python, TypeScript, Go, Java, Ruby, C#, and PHP SDKs all introduce dedicated AnthropicBedrockMantle/BedrockMantleBackend/bedrock.NewMantleClient clients for the Messages-API endpoint; existing AnthropicBedrock/BedrockBackend clients are retained for InvokeModel-based applications.
  • Amazon Bedrock doc restructure — "Claude in Amazon Bedrock" is promoted from research-preview status to the primary integration page; the previous page is renamed "Claude on Amazon Bedrock (legacy)" and the research-preview page is removed.
  • Claude Code npm install clarified — npm installation is no longer labeled deprecated; the package is documented as delivering the native binary via per-platform optional dependencies, with new troubleshooting guidance for missing-binary scenarios.
  • Claude Code hooks once field scoping — The once hook field is now documented as honored only in skill frontmatter; it is explicitly ignored in settings files and agent frontmatter.
  • Model config opusplan context limit — Clarified that the automatic 1M context upgrade does not apply to the opusplan alias; the plan-mode Opus phase runs within the standard 200K context window.

Updated

Added a new troubleshooting section for the "native binary not found after npm install" error, covering disabled optional dependencies, unsupported platforms, and corporate mirror gaps.

  • Native binary not found section — New section documents the Could not find native binary package error and its three root causes: optional dependencies disabled by package manager flags or .npmrc, unsupported platform, and corporate npm mirror missing platform packages.
