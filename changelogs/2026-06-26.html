Key Highlights

  • Model deprecation cleanup — References to claude-opus-4-0, claude-opus-4-20250514, claude-sonnet-4-0, claude-sonnet-4-20250514, and claude-3-haiku-20240307 removed from all API reference model enumerations across 7 SDKs and the REST reference; the supported model count in fallback-hop parameters drops from 17 to 12.
  • BetaFallbackBlockParam.trigger — The BetaFallbackBlockParam object gains a trigger field (BetaFallbackRefusalTrigger) across all beta Messages, Batches, and Count Tokens endpoints; the BetaFallbackBlock response type gains the same field with a new frontier_llm refusal category alongside existing cyber, bio, and reasoning_extraction values.
  • MCP server validation — Agent create and update endpoints now enforce that every entry in mcp_servers must be referenced by an mcp_toolset in tools; unreferenced servers are rejected. Updated across the REST reference and all 7 SDK docs.
  • code_execution_20260521 tool version — A new CodeExecutionTool20260521 type (allowed caller value "code_execution_20260521") added to the allowed_callers enum in Messages, Batches, and Count Tokens across REST and 7-SDK references.
  • New BetaWebhookSessionUpdatedEventData event type — A session.updated webhook event (BetaWebhookSessionUpdatedEventData) added to BetaWebhookEventData across the REST Webhooks reference and all 7 SDK webhook docs.
  • Vertex AI rebranded to Google Cloud — All documentation references to "Vertex AI" updated to "Google Cloud" across 18+ platform and feature docs, including the claude-on-vertex-ai page title itself, which is now "Claude on Google Cloud".

Coordinates and bounding boxes

New

New guide explaining how Claude resizes and pads images before processing, how to interpret returned pixel coordinates for bounding boxes and points, and how to either pre-resize images or rescale coordinates in post-processing to align with the original image.

  • Image resize algorithm — Documents the two-constraint resize rule: edge limit (1568 px standard / 2576 px high-res) and visual token limit (1568 / 4784 tokens); explains that the visual token limit is the most common cause of unexpected resizes for portrait documents.
  • Padding behavior — Explains that Claude pads every image to the next multiple of 28 px; emphasizes that coordinates must be normalized against resized (not padded) dimensions.
  • Pre-resize approach — Provides a reference Python implementation of resized_size() to compute the exact post-resize dimensions for both standard and high-resolution tiers.
  • Rescale approach — Documents the coordinate rescaling formula for cases where pre-resizing is impractical.
  • PDF caveat — Notes that server-side PDF rasterization produces dimensions the caller cannot control, so coordinates cannot be reliably mapped back; recommends rasterizing pages to images manually.
