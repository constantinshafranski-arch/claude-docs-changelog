Key Highlights

  • Webhooks (Beta API + 7 SDKs) — New BetaWebhookEvent type and full webhook domain types added to the Beta API reference, covering session lifecycle events (session.created, session.idled, session.requires_action, etc.) and vault/credential events; corresponding SDK docs added for CLI, C#, Go, Java, Python, Ruby, and TypeScript.
  • Managed Agents webhooks — New guide for subscribing to Managed Agents webhook events (session status, multiagent thread, vault, and outcome evaluation), including endpoint registration, X-Webhook-Signature verification via SDK unwrap(), and delivery behavior.
  • Dreams (Managed Agents) — New Research Preview API for asynchronous memory consolidation: submit a memory store plus up to 100 session transcripts to POST /v1/dreams with dreaming-2026-04-21 beta header; the job deduplicates and reorganizes the store into a new output store.
  • Multiagent sessions — callable_agents field replaced by a multiagent coordinator declaration; Research Preview gate removed; updated delegation patterns (parallelization, specialization, escalation) and revised thread event names (session.thread_status_running/idle/terminated).
  • Plugin URL loading (--plugin-url) — New CLI flag and bare-mode option to fetch a plugin .zip archive from a URL at session startup, enabling CI artifact testing without local unpacking.
  • LLM gateway model discovery — Gateway model discovery is now opt-in via CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1 (previously on by default); minimum version bumped to v2.1.129.

Dreams

New

New Research Preview feature that runs an asynchronous job (POST /v1/dreams, requires dreaming-2026-04-21 beta header) to deduplicate and reorganize an agent memory store using past session transcripts as input, producing a separate output memory store.

  • Dreams API — New POST /v1/dreams endpoint accepting a memory_store input and up to 100 sessions inputs; supported models during preview are claude-opus-4-7 and claude-sonnet-4-6.
  • Asynchronous lifecycle — Dream resource transitions through pending → running → terminal states; output memory store ID appears in outputs[] once running.
  • dreaming-2026-04-21 beta header — Required in addition to managed-agents-2026-04-01; SDKs set both automatically.
  • Non-destructive output — Input store is never modified; the reorganized store is written to a new store ID that can be reviewed or discarded.
  • SDK coverage — Full code examples for curl, CLI, Python, TypeScript, C#, Go, Java, PHP, and Ruby.

Subscribe to webhooks

New

New guide covering how to register HTTPS webhook endpoints, verify X-Webhook-Signature signatures using the SDK unwrap() helper, and handle the full set of session lifecycle, multiagent thread, vault, and outcome evaluation events without polling.

  • Supported event types — Documents session events (session.status_run_started, status_idled, status_rescheduled, status_terminated, thread lifecycle events, outcome_evaluation_ended) and vault/credential events (vault.created/archived/deleted, vault_credential.created/archived/deleted/refresh_failed).

  • Signature verification via unwrap() — SDK unwrap() helper validates the X-Webhook-Signature header and rejects payloads older than 5 minutes; examples provided for Python (Flask), TypeScript (Express), C#, Go, Java, PHP, and Ruby.
  • Delivery behavior — Events carry only type and id; consumers must fetch the full object via a GET call; any non-2xx response triggers a retry.
