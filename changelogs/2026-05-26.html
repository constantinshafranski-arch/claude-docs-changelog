Monitoring

Updated

Clarified OTel event semantics for tool result and tool decision events, tightened `decision_type`/`decision_source` definitions, fixed `user_reject` behavior across interactive vs. non-interactive sessions, and improved headers helper error reporting.

  • Tool result event — emission condition — `claude_code.tool_result` is now explicitly documented as not emitted for rejected tool calls; a cross-reference to the Tool decision event is added for rejections
  • `decision_type` attribute — Always `"accept"` on the tool result event since the event is only emitted after the tool runs
  • `decision_source` attribute — Reject-only sources `"user_abort"` and `"user_reject"` are now explicitly noted as never appearing on the tool result event
  • `"config"` source — Updated to include deny rules (not just allow rules) in the list of sources that produce a `"config"` decision
  • `"user_reject"` source — Clarified that in the interactive CLI, `"user_reject"` is emitted only for the explicit "No" choice; calls matching a personal deny rule emit `"config"` instead; in Agent SDK or non-interactive `-p` sessions, deny-rule matches emit `"user_reject"`
  • OTel headers helper path — The `headerHelper` value now supports executable file paths (including paths with spaces) and shell command lines with arguments; on Windows, the value always runs through the shell
  • OTel headers helper error reporting — Helper failures are now surfaced in `/doctor` output, the debug log (with `--debug` or `/debug`), and stderr in non-interactive `-p` sessions
