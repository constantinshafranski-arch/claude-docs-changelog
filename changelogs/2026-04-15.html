Trigger a routine via API

New

New experimental endpoint POST /v1/claude_code/routines/{routine_id}/fire that starts a Claude Code routine session on demand, returning the session ID and URL; requires a per-routine bearer token and the anthropic-beta: experimental-cc-routine-2026-04-01 header.

  • Endpoint — POST https://api.anthropic.com/v1/claude_code/routines/{routine_id}/fire starts a new run of an existing routine synchronously (returns once the session is created, does not stream output).
  • Authentication — Uses a per-routine bearer token (sk-ant-oat01-…) generated in the Claude Code web UI, not a standard Anthropic API key; token scope is limited to one routine with no read access.
  • Beta header — Every request must include anthropic-beta: experimental-cc-routine-2026-04-01; missing header returns 400 invalid_request_error.
  • Request body — Optional text field (max 65,536 characters) passes freeform context such as an alert body or CI failure log to the routine alongside its saved prompt.
  • Response — Returns {type: "routine_fire", claude_code_session_id, claude_code_session_url} on success (200 OK).
  • Error codes — Documents 400, 401, 403, 404, 429 (with Retry-After), 500, and 503 (note: this endpoint returns 503 where the Claude Platform returns 529 for overloaded errors).
  • SDK support — None; endpoint is Claude Code product surface only, billing against Claude Code subscription rather than Claude Platform API usage.
platform.claude.com/docs/en/api/claude-code/routines-fire →
