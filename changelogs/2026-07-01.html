Key Highlights

  • New `/v1/tunnels` API (beta) — A new first-class Tunnels API at `/v1/tunnels` with `anthropic-beta: mcp-tunnels-2026-06-22` and workspace-scoped WIF tokens supersedes the Admin API tunnel endpoints, which remain available during a migration window.
  • Admin API MCP Tunnels deprecated — All `/v1/organizations/tunnels` Admin API endpoints are now marked deprecated with deprecation notices pointing to the new `/v1/tunnels` beta API.
  • AWS KMS `role_arn` deprecated — The `role_arn` field in external key AWS `provider_config` is now optional and deprecated; Anthropic accesses the KMS key via a managed intermediate role and ignores this field.
  • Analytics: RBAC group null fields — `assigned_seat_count`, `daily_adoption_rate`, `monthly_adoption_rate`, `weekly_adoption_rate`, and `pending_invite_count` now return `null` when the response is scoped to an RBAC group.
  • Analytics users: `bioscience_metrics` and `distinct_connectors_used_count` added — `GET /v1/organizations/analytics/users` now returns a new `bioscience_metrics` object (delegation count, session count, message count, remote compute jobs, skills used) and `distinct_connectors_used_count` in `chat_metrics`.
  • `anthropic-user-profile-id` header added to Messages, Batches, and Count Tokens — All Messages API endpoints (including batch create and count tokens) now accept an `anthropic-user-profile-id` header to attribute requests to a specific user profile, requiring the `user-profiles` beta header.

Tunnels

New

New beta Tunnels API (`/v1/tunnels`) covering CRUD operations, token management, and certificate management for MCP tunnels, requiring `anthropic-beta: mcp-tunnels-2026-06-22` and a WIF token with `workspace:manage_tunnels` scope.

  • POST /v1/tunnels — Creates a tunnel, allocating a fresh hostname; not idempotent; rejects MCP traffic until a CA certificate is added.
  • GET /v1/tunnels/{tunnel_id} — Fetches a tunnel by ID.
  • GET /v1/tunnels — Lists tunnels ordered by creation time; archived tunnels excluded unless `include_archived` is set.
  • POST /v1/tunnels/{tunnel_id}/archive — Irreversibly archives a tunnel, retiring its hostname and invalidating its token.
  • POST /v1/tunnels/{tunnel_id}/reveal_token — Returns the tunnel's current connection token without rotating it.
  • POST /v1/tunnels/{tunnel_id}/rotate_token — Rotates the tunnel token; existing connections are not severed.
platform.claude.com/docs/en/api/beta/tunnels →

Certificates

New

New beta Certificates API under `/v1/tunnels/{tunnel_id}/certificates` for registering, listing, retrieving, and archiving public CA certificates used to authenticate MCP gateway connections.
