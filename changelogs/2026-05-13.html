Use Claude Code on the web

Updated

Clarified that GitHub App installation controls PR webhook access for Auto-fix but does not restrict which repositories a cloud session can reach, and added a toggle for stopping Auto-fix monitoring on a per-PR basis.

  • GitHub auth scope — Clarified that a cloud session can access any repository the connecting GitHub account can see, regardless of which repositories the Claude GitHub App is installed on; App installation enables PR webhooks for Auto-fix only.
  • Access control guidance — Added a Note block explaining that to restrict repository access for cloud sessions, administrators should restrict access on GitHub itself (e.g., team or repository membership), not rely on App installation scope.
  • Auto-fix toggle — Documented that Auto-fix is a per-PR toggle; users can stop monitoring by clearing the Auto-fix toggle in the CI status bar or telling Claude to stop watching the PR.
  • Troubleshooting update — Revised the session creation failure guidance to reflect that the connecting GitHub account's access determines repository reachability, not whether the Claude GitHub App is installed on that specific repository.

Data usage

Updated

Expanded feedback survey suppression rules to include DO_NOT_TRACK and added a new CLAUDE_CODE_ENABLE_FEEDBACK_SURVEY_FOR_OTEL=1 opt-in for organizations routing surveys through their own OpenTelemetry collector.

  • Survey suppression — The feedback survey is now also disabled when DO_NOT_TRACK is set, in addition to the existing DISABLE_TELEMETRY and CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC variables.
  • CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC — Organizations with this flag set now also never see the transcript-share follow-up prompt, consistent with other nonessential traffic suppression.
  • CLAUDE_CODE_ENABLE_FEEDBACK_SURVEY_FOR_OTEL — New env var lets organizations that block nonessential Anthropic traffic still capture session quality ratings via a configured OpenTelemetry collector; transcript-share and all other Anthropic-bound feedback remain disabled.

Output styles

Updated

Rewrote the comparison sections to give clearer decision guidance on when to use output styles versus CLAUDE.md, --append-system-prompt, and subagents.

  • Output styles vs. CLAUDE.md vs. --append-system-prompt — Prose rewritten to lead with the use-case decision: use output styles when Claude should adopt a different identity (e.g., writing editor); use CLAUDE.md or --append-system-prompt when Claude should remain a coding assistant with additional rules.
  • Mechanism differences — Added explicit clarification that output styles edit the system prompt directly, CLAUDE.md adds content as a user message after the system prompt, and --append-system-prompt appends without removing anything.
  • Output styles vs. subagents — Section rewritten to clarify: use an output style to change how the main conversation responds in every session; use a subagent when you need a separately scoped helper that the main conversation delegates to.

Claude Code settings

Updated

Added three new settings: teammateDefaultModel for agent teams, per-marketplace autoUpdate, and parentSettingsBehavior for embedding hosts; also clarified worktree.sparsePaths behavior.

  • teammateDefaultModel — New user setting (stored in ~/.claude.json) that sets the default model for agent team teammates when the spawn prompt does not specify one; accepts a model alias such as "sonnet" or null to inherit the lead's current model.
  • autoUpdate per marketplace — Marketplace entries in settings now accept an optional autoUpdate Boolean; official Anthropic marketplaces default to true, all others default to false.
  • parentSettingsBehavior — New setting allowing embedding hosts (e.g., Claude Desktop) to supply policy via the SDK managedSettings option; set to "merge" so embedder values are considered but can only tighten, not loosen, managed policy.
  • worktree.sparsePaths — Clarified that sparse-checkout writes only the listed directories plus root-level files to disk (cone mode detail removed from description).
