Key Highlights

  • Claude Code Desktop: Browser pane — The preview pane is now a full tabbed Browser pane supporting external site navigation, per-site tool approval, and a new browserExternalPageTools managed setting to restrict Claude's actions on external pages.
  • Claude Code CLI: New claude doctor command &amp; --append-subagent-system-prompt flag — Added claude doctor for read-only installation diagnostics from the shell, and --append-subagent-system-prompt (v2.1.205+) to inject text into every subagent's system prompt in non-interactive mode.
  • Managed Agents: agent-memory-2026-07-22 beta header — Memory store endpoints now require a dedicated agent-memory-2026-07-22 beta header (separate from managed-agents-2026-04-01); the list memories API also gained cleaner path_prefix and depth semantics.
  • Managed Agents: Custom tools from self-hosted sandboxes — Self-hosted sandbox workers can now serve custom tools directly, including wrapping an internal MCP server as custom tools without requiring inbound connectivity from Anthropic.
  • Authentication: API key expiration — API keys now support configurable expiration (preset or custom durations) at creation time, with expiration visible in the Admin API expires_at field and email warnings before expiry.
  • API and data retention: Major doc restructure — The ZDR and HIPAA readiness sections were substantially rewritten, clarifying scope, adding explicit HIPAA error handling guidance, and introducing a step-by-step guide for enabling 30-day retention per workspace.

Updated

Introduced the dedicated agent-memory-2026-07-22 beta header for all memory store endpoints, clarified list memories path_prefix and depth semantics, dropped order_by from examples, and added a warning about combining both headers.

Authentication

Updated

Added a new Key expiration section documenting expiration presets, admin policy enforcement, email warnings, and Admin API expires_at field.

  • Key expiration section — New section documents expiration choices at key creation (3 hours, 1 day, 7 days, 30 days, custom, or Never); org-level maximum policies limit available options and remove Never.
  • Expiration email warnings — Anthropic emails the key creator 7 days before expiry (keys ≥14 day lifetime) or 1 day before (keys ≥7 day lifetime); shorter-lived keys expire without a warning email.
  • Post-expiration behavior — Expired keys return 401 authentication_error; they cannot be reactivated.
