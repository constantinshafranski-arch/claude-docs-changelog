Key Highlights

  • agent-memory-2026-07-22 beta flag — A new "agent-memory-2026-07-22" beta version string has been added to the AnthropicBeta enum, now listing 26 recognized values across all beta-enabled endpoints.
  • Custom tool description limit — The description field for custom tools on Agent Create and Agent Update endpoints has been expanded from 1–1024 to 1–4096 characters, enabling richer tool descriptions.
  • BetaSelfHostedWork.secret field removed — The secret field (credential payload for environment workers) has been dropped from the BetaSelfHostedWork response object across all Work endpoints (get, ack, poll, list, stop, update); the object now exposes 9 rather than 10 additional fields.
  • Pervasive beta header propagation — The agent-memory-2026-07-22 beta value has been added to 300+ API reference pages spanning all SDK languages (Python, TypeScript, Go, Java, Ruby, C#, PHP, CLI) and all resource types (agents, deployments, deployment runs, environments, files, memory stores, messages, sessions, skills, tunnels, user profiles, vaults, webhooks).
  • High doc volume — Due to the large number of updated reference files (300+), individual per-language SDK entries are consolidated; changes are uniform across all languages.
  • agent-memory-2026-07-22 beta header — A new beta flag has been added across all API endpoints (Files, Memory Stores, Messages, Batches, Models, Sessions, and more), replacing the former managed-agents-2026-04-01 header for memory store operations.
