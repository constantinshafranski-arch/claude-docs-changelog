Key Highlights

  • New analytics endpoints — Two new Admin API endpoints added: GET /v1/organizations/analytics/artifacts for artifact-creation activity by MIME type, and GET /v1/organizations/analytics/plugins for per-plugin install and invocation usage across Cowork and Claude Code.
  • Analytics data lag reduced to 1 day — All analytics endpoints (summaries, skills, connectors, users, chat projects, cost, usage) updated from a 3-day finalization lag to a 1-day typical lag; default date ranges and availability semantics updated accordingly.
  • Analytics filtering and grouping expanded — Analytics endpoints now accept filter[] (supports rbac_group_id), group_by[], order, order_by, and ending_date/starting_date range-rollup mode across skills, connectors, chat projects, users, and plugins.
  • claude-sonnet-5 added to model enums — The new claude-sonnet-5 model (high-performance model for coding and agents) is now surfaced in the BetaManagedAgentsModel enum and all Messages/Sessions API model lists.
  • Sessions API: agent overrides and streaming deltas — Create Session now accepts a BetaManagedAgentsAgentWithOverridesParams agent type for per-session MCP server overrides; Stream Events gains an event_deltas query parameter for previewing agent.message and agent.thinking content before the final event.
  • Self-hosted work items gain a secret field — BetaSelfHostedWork objects now include a secret field (credential payload for the environment worker), populated on poll/ack responses and null on all other retrieval paths.
