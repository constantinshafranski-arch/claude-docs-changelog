Key Highlights

  • SDK docs path migration — All references to /docs/en/api/client-sdks have been updated to /docs/en/cli-sdks-libraries/overview across platform integration guides (Bedrock, Vertex AI, Foundry, AWS), authentication, and streaming docs.
  • Skills disallowed-tools — Claude Code Skills now support a disallowed-tools frontmatter field to restrict Claude's available tools for the duration of a single skill activation, with scope and lifetime clarified.
  • Task budget visibility — New note clarifies the budget countdown is model-only; API responses and SDKs expose no remaining-budget field, requiring client-side summation of usage across loop turns.
  • Zero Data Retention eligibility — ZDR for Claude Code on Enterprise is now documented as requiring eligibility confirmation; it is not included in the standard Enterprise plan.
  • Advisor tool generic properties — The advisor tool now explicitly accepts cache_control, allowed_callers, defer_loading, and strict tool definition properties.
  • Dreams cancellation usage — New note warns that usage fields on a canceled dream may continue updating for seconds after cancellation; callers must poll until values stabilize.
