Key Highlights

  • Dev containers overhaul — The development containers doc was fully rewritten with step-by-step setup via the Claude Code Dev Container Feature, sections on persistent auth volumes, organization policy enforcement, network egress restriction, and running without permission prompts.
  • Thinking block retention defaults — Context editing, adaptive thinking, extended thinking, and prompt caching docs updated to reflect model-class-specific defaults: Opus 4.5+ and Sonnet 4.6+ keep all prior thinking blocks; earlier Opus/Sonnet and all Haiku models keep only the last turn.
  • MCP alwaysLoad flag — New alwaysLoad field (v2.1.121+) lets individual MCP servers or tools bypass tool-search deferral and load into context at session start; initial connection retry logic with exponential backoff also documented.
  • Hooks updatedToolOutput field — PostToolUse hooks gain a new updatedToolOutput field that replaces built-in tool output seen by Claude (superseding the MCP-only updatedMCPToolOutput), with schema-validation semantics documented.
  • CLI claude ultrareview command — New claude ultrareview [target] command added to the CLI reference for running ultrareview non-interactively with --json and --timeout flags.
  • SDK type assertion removal — Multiple TypeScript SDK examples across adaptive thinking, extended thinking, compaction, and context editing docs dropped as unknown as … type assertions, and C#/Go SDK examples for display: omitted now use native SDK types instead of raw HTTP requests.

Connect Claude Code to tools via MCP

Updated

Documented initial-connection retry behavior for HTTP/SSE MCP servers and the new alwaysLoad field for exempting servers from tool-search deferral.

  • Initial connection retry (v2.1.121+) — HTTP/SSE servers that fail their initial connection are now retried up to three times on transient errors (5xx, connection refused, timeout); authentication and not-found errors are not retried.
  • alwaysLoad server flag (v2.1.121+) — Setting alwaysLoad: true in a server's MCP config causes all its tools to load into context at session start, bypassing tool-search deferral; also available per-tool via "anthropic/alwaysLoad": true in the tool's _meta object.
