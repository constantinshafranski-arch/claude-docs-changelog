Checkpointing

Updated

Added a new section documenting that edits made by background subagents and background forked skills fall outside session checkpoints and cannot be restored by rewinding.

  • Subagent edits not restored — Clarified that edits from background subagents (including /code-review --fix and background forked skills) are not tracked by session checkpoints; only foreground forked skills (background: false) have their edits restored on rewind
  • Foreground fork exception — Documented that a skill with context: fork running in the foreground edits the working tree during the invoking turn, so rewinding restores those edits as usual
  • Version note — Forked skills ran in the foreground by default before v2.1.218; from v2.1.218 onward they default to background

Enterprise network configuration

Updated

Documented that proxy and TLS certificate environment variables are read only from managed settings and ~/.claude/settings.json in Claude Desktop sessions where the app manages the provider connection, preventing checked-out repositories from redirecting those paths.

  • Proxy variable scoping in Claude Desktop — In Claude Desktop sessions where the app manages the provider connection (Code tab on a third-party provider, Cowork sessions), HTTP_PROXY, HTTPS_PROXY, NO_PROXY, and TLS certificate variables are read only from managed settings and ~/.claude/settings.json; repository-level settings files are ignored (v2.1.217)
  • Local and SSH session behavior — Local, SSH, and WSL Code tab sessions signed in through claude.ai are not app-managed, so Claude Code reads these variables from every settings scope as in any terminal session
  • Pre-v2.1.217 behavior — Before v2.1.217, Claude Code ignored these variables in every settings file when the app managed the connection

Extend Claude with skills

Updated

Expanded the forked-subagent section with background execution controls, a new background frontmatter field, updated stacking behavior, and clarifications on tool availability and checkpoint scope for backgrounded forks.

  • background frontmatter field — New optional field (v2.1.218) on skills with context: fork; defaults to true (runs in background); set to false to block the invoking turn until the forked subagent completes
  • Background fork forced-foreground cases — Even without background: false, forked skills run in the foreground in non-interactive (-p) mode, when CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1, when the same skill is already running, or when fired by a scheduled task
  • Tool set for backgrounded forks — Background forked skills use the narrower background-subagent tool set; set background: false to retain the full tool set
  • Checkpoint scope — Edits from backgrounded forked skills fall outside session checkpoints and cannot be undone with /rewind; use git to revert
  • Boolean frontmatter values — Skill frontmatter Boolean fields now accept yes, no, on, off, 1, and 0 in any case (v2.1.218)
  • /code-review stacking behavior — /code-review now runs as a forked subagent from v2.1.218, ending skill-stack expansion at that token; on earlier versions it ran inline and could be stacked
