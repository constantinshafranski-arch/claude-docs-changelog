Release Notes

Key Highlights

  • Dreams API (new) — Six new beta endpoints cover the full lifecycle of asynchronous dreaming jobs: create, list, retrieve, cancel, and archive, with token-usage stats tracked across pipeline stages; requires the dreaming-2026-04-21 beta header.
  • Effort field on agents and sessions — BetaManagedAgentsModelConfigParams gained an effort field accepting levels low/medium/high/xhigh/max across all agent and session CRUD endpoints, and both agents and sessions propagate the level to every Messages call they make.
  • Thinking documentation restructured — The single adaptive-thinking page was replaced by four focused docs: thinking, thinking-steering-and-cost, thinking-tool-workflows, and thinking-troubleshooting; all model and build pages updated their cross-references accordingly.
  • New general_harms refusal category — The stop_details.category enum on Messages, Message Batches, and batch results grew a fifth value "general_harms", and each category now carries an inline prose description.
  • Self-hosted work items gain a secret field — BetaSelfHostedWork now returns a secret credential payload when polling for work; the field is null on all other retrieval paths, and the response object grew from 9 to 10 fields.
  • Claude Code: new Checkpointing doc and emoji shortcodes — A new Checkpointing page documents automatic snapshot tracking, the /rewind menu, and summarize-from/up-to actions; v2.1.217 also added :emoji-shortcode: support in the prompt input.

Checkpointing

New

New doc covering Claude Code's automatic checkpoint system: every user prompt snapshots file state, the /rewind menu lets you restore code, conversation, or both, and two new summarize actions compress selected portions of the conversation while preserving context.

  • Automatic tracking — Every user prompt creates a checkpoint; Claude Code keeps snapshots for the 100 most recent checkpoints per session and cleans them up after 30 days.
  • /rewind menu — Opened with /rewind or double-Esc on an empty input; options include restore code, restore conversation, restore both, summarize-from-here, and summarize-up-to-here.
  • Summarize vs restore — Restore reverts state; summarize compresses part of the conversation history into an AI-generated summary without changing files on disk.
  • Rewind past /clear — A /resume &lt;session-id&gt; entry appears at the top of the rewind menu when you previously cleared the conversation (requires v2.1.191+).
code.claude.com/docs/en/checkpointing →

Updated

Expanded deployment initial event support to user.define_outcome, refined jitter semantics, and clarified archival behavior when an agent is deleted versus archived.

  • Initial event requirement — Deployments now require at least one initial event, either a user.message or user.define_outcome, instead of only user.message.

  • Agent deletion vs. archival — When an agent is archived, the deployment is archived in the same operation; when the agent is deleted, the next scheduled trigger detects the missing agent and archives the deployment.
