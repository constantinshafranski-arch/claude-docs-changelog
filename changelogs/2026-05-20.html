Key Highlights

  • MCP Tunnels API — New admin API resource (beta `mcp-tunnels-2026-05-19`) with full CRUD, token reveal/rotate, and certificate management endpoints for self-hosted MCP gateway tunnels.
  • Self-hosted sandbox Work API — New `/v1/environments/{id}/work` endpoint family covering poll, ack, heartbeat, stop, list, update, and stats operations, enabling orchestration of sessions against self-hosted sandbox environments; covered across all 7 SDKs plus CLI.
  • Environments self-hosted type — The `config` field on Environments endpoints now accepts `BetaSelfHostedConfigParams` in addition to `BetaCloudConfigParams`, and the `BetaEnvironment` response type was extended with a new field to support self-hosted configurations.
  • Workspace tags — Workspace create, update, list, retrieve, and archive endpoints now expose a `tags: map[string]` field for user-defined key-value metadata; workspace update also gained an `anthropic-beta` header and made `name` optional.
  • AnthropicBeta enum — Added `"cache-diagnosis-2026-04-07"` to the `AnthropicBeta` union, reflected across the Beta domain types and all agent/environment API endpoints in every SDK.
  • Volume note — This release touched 249 updated and 39 new API reference docs across Python, TypeScript, Go, Java, Ruby, C#, and CLI SDKs; individual per-SDK entries are omitted due to volume but mirror the changes described above.

MCP Tunnels

New

New admin API resource providing full lifecycle management of MCP tunnels including retrieve, list, archive, token reveal/rotate, and X.509 certificate management; all endpoints require the `mcp-tunnels-2026-05-19` beta header.

  • GET /v1/organizations/tunnels/{tunnel_id} — Retrieve a single tunnel by ID, returning id, archived_at, created_at, display_name, hostname, and workspace_id
  • GET /v1/organizations/tunnels — List tunnels org-wide, ordered by creation time; supports `workspace_id`, `include_archived`, `limit`, and `page` query parameters
  • POST /v1/organizations/tunnels/{tunnel_id}/archive — Irreversibly archive a tunnel, invalidating its token and retiring its hostname; co-archives all non-archived certificates
  • POST /v1/organizations/tunnels/{tunnel_id}/reveal_token — Return the tunnel's current connection token without storing it; exposed as POST to avoid logging in access logs
  • POST /v1/organizations/tunnels/{tunnel_id}/rotate_token — Invalidate the current token and return a fresh value; established connections are not severed; accepts optional `reason` for audit
  • POST /v1/organizations/tunnels/{tunnel_id}/certificates — Register a PEM-encoded X.509 CA certificate for TLS verification; tunnel holds at most two non-archived certificates
  • GET /v1/organizations/tunnels/{tunnel_id}/certificates/{certificate_id} — Retrieve a single registered certificate by ID
  • GET /v1/organizations/tunnels/{tunnel_id}/certificates — List certificates on a tunnel; archived excluded by default
  • POST /v1/organizations/tunnels/{tunnel_id}/certificates/{certificate_id}/archive — Archive a certificate, removing it from the trusted set; archiving the last certificate causes the tunnel to reject MCP traffic
platform.claude.com/docs/en/api/admin/mcp_tunnels →

Work

New

New environment Work API covers the full worker protocol for self-hosted sandbox environments: retrieve, poll (long-poll), ack, heartbeat, stop, list, update, and stats endpoints; automatically called by pre-built SDK/CLI workers but documented for reference.

  • GET /v1/environments/{id}/work/{work_id} — Retrieve detailed information about a specific work item
  • GET /v1/environments/{id}/work/poll — Long-poll for queued work items; `block_ms` controls wait time (1–999ms); `reclaim_older_than_ms` reclaims stale unacknowledged items
  • POST /v1/environments/{id}/work/{work_id}/ack — Acknowledge receipt of a work item, transitioning it from `queued` to `starting`
  • POST /v1/environments/{id}/work/{work_id}/heartbeat — Record a heartbeat to maintain the lease; supports `desired_ttl_seconds` and optimistic-concurrency via `expected_last_heartbeat`
  • POST /v1/environments/{id}/work/{work_id}/stop — Stop a running work item
  • GET /v1/environments/{id}/work — List work items in the environment with pagination
  • POST /v1/environments/{id}/work/{work_id} — Update a work item's status or metadata
  • GET /v1/environments/{id}/work/stats — Retrieve aggregate statistics for the environment's work queue
platform.claude.com/docs/en/api/beta/environments/work →
