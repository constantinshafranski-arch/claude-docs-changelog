Bash tool

Updated

Major rewrite clarifying that the bash tool is a client-executed tool, adding a tool versioning section, providing a full sentinel-based implementation example, and cleaning up SDK code snippets by removing redundant import statements.

  • Execution model — Reframed introduction to make clear that Claude emits a tool_use block with the command to run; the application owns the shell process, not Claude.
  • Tool versions — New section documenting bash_20250124 (current, no beta header, all models from Sonnet 3.7+) vs. bash_20241022 (computer-use beta, October 2024 Sonnet 3.5 only).
  • JSON output example — Added annotated stop_reason: tool_use response block showing the tool_use content structure returned by Claude.
  • Parameters section — Clarified tool definition is schema-less, added description of how to handle restart: true by killing and restarting the shell process.
  • Implementation guide — Replaced abstract steps with a concrete sentinel-based BashSession implementation in Python and TypeScript using start_new_session=True for process-group isolation.
  • SDK snippets — Removed redundant top-level import statements from all language examples (Python, TypeScript, C#, Go, Java, PHP, Ruby).

Hooks reference

Updated

Added a bug-fix note for async hooks: malformed or type-invalid JSON output fields are now dropped with a --debug warning instead of crashing the session (fixed in v2.1.202).

  • Async hook output validation — Claude Code now validates async hook JSON responses against the same output schema as synchronous hooks; fields with wrong types (e.g., a non-string systemMessage) are silently dropped and flagged in --debug output.
  • Session crash fix — Before v2.1.202, malformed JSON from an async hook could crash and permanently break session resumption; this is resolved.
