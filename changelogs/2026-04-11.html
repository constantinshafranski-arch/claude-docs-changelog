Release Notes

Key Highlights

  • New doc: Claude in Amazon Bedrock (research preview) — Full guide for the new Messages API endpoint at /anthropic/v1/messages on AWS-managed infrastructure with zero operator access, available in us-east-1 via invite; covers Bedrock service role, IAM assumed role, and bearer token auth paths across Python, TypeScript, Go, Java, C#, PHP, and Ruby SDKs.
  • Code execution tool model matrix — Model compatibility table expanded to explicitly list supported models per tool version; code_execution_20260120 (REPL state persistence + programmatic tool calling) is now documented as limited to Opus 4.5+ and Sonnet 4.5+, while code_execution_20250825 covers a broader set including Haiku 4.5 and Opus 4.1.
  • Enterprise network config: CA certificate store control — New CLAUDE_CODE_CERT_STORE environment variable added, accepting comma-separated bundled and/or system values to control which CA roots Claude Code trusts; documents that TLS-inspection proxies (CrowdStrike Falcon, Zscaler) work automatically when their root is in the OS store.
  • Structured outputs: PHP class-based helper and TypeScript jsonSchemaOutputFormat() — PHP SDK now supports StructuredOutputModel interface with outputConfig: ['format' => MyClass::class]; TypeScript SDK adds jsonSchemaOutputFormat() helper with compile-time type inference from as const JSON Schema literals.
  • Homebrew two-cask split — claude-code now tracks the stable release channel (~1 week behind) and claude-code@latest tracks the latest channel; documented across Overview, Quickstart, Setup, and Troubleshooting pages.
  • allowManagedHooksOnly exemption for force-enabled plugins — Hooks from plugins force-enabled via managed settings enabledPlugins are now exempt from the allowManagedHooksOnly block, enabling administrators to distribute vetted hooks through an organization marketplace.

Claude in Amazon Bedrock (research preview)

New

New guide for the Messages API research preview on Amazon Bedrock: covers the /anthropic/v1/messages endpoint with AWS-native SigV4 auth and SSE streaming, three authentication paths (Bedrock service role, IAM assumed role, bearer token), SDK installation for Python/TypeScript/Go/Java/C#/PHP/Ruby, and feature availability, quotas, regions, and observability details.

  • New endpoint — Documents https://bedrock-mantle.{region}.api.aws/anthropic/v1/messages as the research preview endpoint, using the same request shape as the first-party Claude API with zero operator access (Anthropic has no access to inference infrastructure).
  • Authentication options — Three paths documented: Bedrock service role (recommended, AWS-managed keys), IAM assumed roles (12-hour max session, identity-federated), and bearer tokens (aws-bedrock-token-generator CLI, 12-hour max, least preferred).
  • SDK support — Installation and code samples for all official SDKs using AnthropicBedrockMantle client classes; Python (anthropic[bedrock]), TypeScript (@anthropic-ai/bedrock-sdk), C# (Anthropic.Bedrock), Go (bedrock.NewMantleClient), Java (BedrockMantleBackend), PHP, and Ruby.
  • Access requirements — Research preview requires a dedicated AWS account in us-east-1; account ID must be submitted by an Anthropic account executive for allowlisting (typically 24-hour turnaround).
platform.claude.com/docs/en/build-with-claude/claude-in-amazon-bedrock-research-preview →

Structured outputs

Updated

Added the TypeScript jsonSchemaOutputFormat() helper with compile-time type inference, rewrote PHP examples to use the new StructuredOutputModel interface, updated several SDK code samples, and revised the helper-method note to accurately scope which SDKs accept output_format as a convenience parameter.

  • TypeScript jsonSchemaOutputFormat() — New helper in @anthropic-ai/sdk/helpers/json-schema accepts a JSON Schema object; when called with an as const literal, parsed_output is compile-time typed to match the schema; without as const, type collapses to unknown.   • Language support table — Added PHP to the native schema helpers list and moved C#, Go, and CLI to the raw JSON schema group.
  • SDK code cleanup — Simplified TypeScript client instantiation (new Anthropic()), updated PHP client instantiation (new Client()), used Model.ClaudeOpus4_6 enum in C# example, replaced map[string]interface{} with map[string]any in Go, and updated C# content extraction to use TryPickText.
