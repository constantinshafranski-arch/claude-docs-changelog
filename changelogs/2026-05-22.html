Key Highlights

  • MCP proxy service renamed — The mcp-gateway service and its config file have been renamed to mcp-proxy and mcp-proxy.yaml across all Docker Compose and Helm deployment docs; update any existing deployments and log-monitoring scripts accordingly
  • Auto mode availability expanded — Auto mode is now available to all users on the Anthropic API (not just Max/Team/Enterprise plans), and requires Claude Sonnet 4.6, Opus 4.6, or Opus 4.7
  • MCP per-server tool timeout — A timeout field (milliseconds) can now be set per server in .mcp.json to cap tool call wall-clock time, overriding MCP_TOOL_TIMEOUT for that server; values below 1000 ms are floored to 1 second
  • Sandboxing doc overhauled — The sandboxing page was rewritten with a step-by-step quickstart, explicit escape-hatch behavior (dangerouslyDisableSandbox), a strict mode toggle (allowUnsandboxedCommands: false), and clarified network/filesystem isolation defaults
  • CLAUDE_CODE_SUBAGENT_MODEL now covers agent teams — The env var now applies to both subagents and agent teams, and supports inherit to fall back to normal model resolution
  • Managed MCP configuration extracted to dedicated page — The large managed MCP section (Option 1 managed-mcp.json and Option 2 allowlist/denylist) was removed from the MCP doc and now lives at /en/managed-mcp; all cross-references updated accordingly

Configure the sandboxed Bash tool

Updated

The sandboxing page was substantially rewritten with a step-by-step quickstart, explicit escape-hatch and strict-mode documentation, and reorganized sections covering filesystem/network isolation behavior.

  • Page restructured — Overhauled from a conceptual overview into a task-oriented guide with Get started steps, Configure sandboxing, and How sandboxing works sections
  • Quickstart via /sandbox — Added a three-step guide using the /sandbox command to enable the sandbox and choose approval mode
  • Escape hatch documented — Explicitly described the dangerouslyDisableSandbox parameter behavior: Claude may retry failed sandboxed commands outside the sandbox through the normal permission flow
  • Strict sandbox mode — Documented allowUnsandboxedCommands: false setting (exposed as Strict sandbox mode in the /sandbox Overrides tab) to fully disable the escape hatch
  • Linux/WSL2 dependency check — The Dependencies tab in /sandbox now shows status of bubblewrap, socat, ripgrep, and the optional seccomp filter; AppArmor and WSL notes moved to collapsible accordions
  • Network isolation defaults clarified — No domains are pre-allowed; first-use prompts approval; allowManagedDomainsOnly blocks non-allowed domains automatically
  • Filesystem defaults clarified — Default read access still reaches credential files like ~/.aws/credentials and ~/.ssh/; added explicit recommendation to add them to denyRead
