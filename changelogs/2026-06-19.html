Key Highlights

  • New activity types in Compliance API — 11 new filterable event types added across activity_types and exclude_activity_types, including abuse_decision_received, claude_project_document_updated, claude_project_sync_source_*, design_project_*, org_capability_grant_*, and org_data_residency_updated.
  • New code_execution_20260521 tool version — Discloses the 90-second per-cell wall-clock time limit in the tool description so Claude can budget long-running cells; added to the supported versions table for all compatible models.
  • New web search and web fetch tool versions — web_search_20260318 and web_fetch_20260318 add a response_inclusion parameter to exclude consumed server-tool result blocks from API responses, reducing output token costs in agentic workflows; web_fetch_20260309 adds cache-bypass control via use_cache: false.
  • New Admin API key guide — Dedicated doc covering creation steps, key prefixes, and scope selection for both Claude Console (sk-ant-admin01-) and Claude Enterprise (sk-ant-api01-) organizations, with a full scope table for Compliance, Analytics, and Spend Limits APIs.
  • Compliance API file metadata enriched — files and generated_files objects on chat messages and project attachments now include md5, size_bytes, and created_at fields across the Apps, Chats, Messages, and Projects endpoints.
  • Organization settings now return api_keys array — GET /v1/compliance/organizations/{id}/settings response includes a new api_keys field listing all Compliance API keys for the hierarchy with id, is_active, scopes, and creator metadata.

Updated

Expanded the claude_code.api_refusal OpenTelemetry event with nine new attributes covering query source, speed, retry attempt, effort, server fallback, refusal category, and skill/plugin attribution.

  • query_source — Subsystem that issued the refused request (e.g., repl_main_thread, compact).
  • speed — "fast" or "normal" depending on Fast mode state.
  • attempt — Retry attempt number (1-indexed).
  • effort — Effort level applied to the request; absent when model does not support effort.
  • server_fallback_hop — true when a server-side model fallback retried the refusal; false when the request ended in a refusal visible to the user.
  • has_category / category — Indicates whether and what refusal category was returned (cyber, bio, frontier_llm, reasoning_extraction); category only present with OTEL_LOG_TOOL_DETAILS=1.
  • has_explanation — Whether the API response carried a stop_details.explanation.
