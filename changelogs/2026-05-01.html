Release Notes

Key Highlights

  • User Profiles API (beta) — New user-profiles-2026-03-24 beta flag and full CRUD endpoints (/v1/user_profiles) added across all 7 SDK languages and the CLI, including enrollment URL generation.
  • BetaEnvironment terminology — Environment resource descriptions updated from "cloud and BYOC environments" to "cloud and self-hosted environments" across all Create, Retrieve, Archive, and Update endpoints.
  • Messages max_tokens: 0 support — The Messages and Message Batches max_tokens parameter now explicitly supports 0 to populate the prompt cache without generating a response.
  • BetaCompactionBlock encrypted_content field — Compaction block schema extended with an encrypted_content field for round-tripping opaque compaction metadata, reflected in Messages and Batch Results responses.
  • Memory Store & Memory API documentation overhaul — Operation names normalized (e.g., CreateMemoryStore → "Create a memory store"), field descriptions substantially expanded for name, description, metadata, content, path, precondition, and response types across all memory store endpoints.
  • 206 API reference docs updated, 12 new docs added — The AnthropicBeta union now enumerates 21 members (previously 20) across all beta-capable endpoints; many lower-volume SDK-specific page entries were not individually listed.
