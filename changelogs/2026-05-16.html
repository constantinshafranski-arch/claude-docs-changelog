Parallel tool use

Updated

Added a new "Execution semantics" section documenting that tool calls within a single assistant turn are unordered, can run concurrently, and that dependent-call failures should be returned with is_error: true so Claude can reissue; also added a corresponding troubleshooting item.

  • Execution semantics (new section) — Documents that tool calls in one assistant turn are unordered and can be dispatched with Promise.all / asyncio.gather; Claude issues dependent calls across separate turns.
  • Dependent-call error handling — Instructs callers to return is_error: true with the natural error message when a batched call fails due to a missing prerequisite; Claude recovers and reissues rather than requiring the caller to detect dependencies in advance.
  • Troubleshooting item #4 — Added guidance: do not switch to sequential execution when batch calls appear dependent; instead return the error and optionally add "Only batch tool calls that are independent of each other" to the system prompt.
  • disable_parallel_tool_use formatting — Fixed inline code formatting on tool_choice references in the disable instructions.

  • C# example — Added full BetaRunnableTool usage with Definition + Run delegate pattern and InputSchema JSON schema construction.

  • Java example — Added Supplier&lt;String&gt;-based tool class pattern with Jackson annotation-driven schema derivation.
  • @beta_tool decorator note — Simplified Python decorator explanation: removed the inline JSON schema example, replaced with a one-line description of schema derivation.
  • Compaction Tip removed — Removed the automatic context management / compaction Tip block from the introduction.

Data usage

Updated

Updated /feedback command documentation to describe session-scoped history selection and added behavior for third-party provider environments where feedback is written to a local archive instead of sent to Anthropic.

  • /feedback history selection — Users now choose how much history to include before submitting: current session only (default), or sessions from the same project over the last 24 hours or 7 days.
