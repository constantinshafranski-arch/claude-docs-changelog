Key Highlights

  • Claude Opus 4.1 deprecated — Claude Opus 4.1 is now marked as deprecated across all documentation, including tool compatibility tables, pricing tables, platform model listings (Bedrock, Vertex AI, Foundry), and feature support references.
  • Claude Code settings: requiredMinimumVersion / requiredMaximumVersion — Two new managed-settings-only keys enforce a hard version range at startup; processes outside the range exit immediately, unlike the soft minimumVersion floor which only constrains updates.
  • Hooks reference: if-field Bash matching table — A new decision table documents exactly when the if filter fires for subcommands, $(), backticks, and $VAR expansions, and clarifies fail-open behavior; additionalContext is now supported on SubagentStop hooks.
  • Monitoring: new api_refusal OTEL event — A dedicated claude_code.api_refusal event is emitted when a response stream ends with stop_reason: "refusal", enabling refusal-frequency tracking; MCP server connection events also gain is_plugin, plugin_id_hash, and plugin.name attributes.
  • Claude Code headless: background-task exit behavior — Background Bash tasks started during a claude -p run are now terminated ~5 seconds after the final result, fixing a pre-v2.1.163 hang when a never-exiting process held the invocation open.
  • Model alias resolution clarified for Bedrock, Vertex AI, and Foundry — Documentation across all three cloud-provider guides now states that unresolved aliases fall back to Claude Code's built-in provider default (which can lag the latest release), not always the newest Anthropic release.

Claude Code settings

Updated

Two new managed-settings keys (requiredMinimumVersion and requiredMaximumVersion) were added to enforce a hard startup version range, and the minimumVersion description was updated to reference the new hard-floor option; default commit attribution format also changed.

  • requiredMinimumVersion — New managed-settings-only key; if the running Claude Code version is below this floor, the process exits at startup and instructs the user to update. Unlike minimumVersion, this blocks startup rather than only constraining updates.
  • requiredMaximumVersion — New managed-settings-only key; if the running version exceeds this ceiling, the process exits at startup; auto-updates and claude update skip versions above the ceiling. Recovery tools (claude update, claude install, claude doctor) remain functional above the ceiling.
  • minimumVersion description — Added a cross-reference to requiredMinimumVersion for cases where a hard startup block is needed rather than a soft update floor.
  • Default commit attribution — Removed the 🤖 emoji and [Claude Code](https://claude.com/claude-code) line from the default commit trailer format, leaving only the Co-Authored-By trailer; added a note that the model name reflects the active session model.
