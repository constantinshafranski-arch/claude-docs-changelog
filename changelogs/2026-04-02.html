Hooks reference

Updated

Comprehensive updates to hook system including new PermissionDenied event, defer mechanism for non-interactive mode, and output length limits.

  • PermissionDenied hook documentation — New hook event fires in auto mode when classifier denies tool calls; input includes tool_name, tool_input, tool_use_id, and reason
  • Defer tool calls for later — PreToolUse hooks can return permissionDecision: "defer" in -p mode to pause and let calling process resume with answer via Agent SDK
  • Deferred tool use payload — Includes id, name, and input fields; session remains on disk until resumed; exit with stop_reason: "tool_deferred"
  • PreToolUse decision values — Expanded from three (allow/deny/ask) to four with "defer" option for non-interactive integration
  • Hook output length limit — Output injected into context capped at 10,000 characters; excess saved to file with preview and path
  • Tool lifecycle diagram — Updated diagram shows PermissionDenied as side branch from PermissionRequest for auto-mode denials
  • Exit code 2 behavior table — PermissionDenied row added showing exit code ignored; use JSON hookSpecificOutput.retry: true instead
