Key Highlights

  • Organization default model & effort limits — Claude Code v2.1.196+ Enterprise admins can now set a default model per role and cap effort levels per model per role; the default alias resolves to the org default, and the /model picker surfaces it as "Org default".
  • Sandbox credential masking — New credentials.envVars[].mode: "mask" in Claude Code v2.1.199 replaces a sandboxed variable with a per-session sentinel and injects the real credential at the proxy layer for listed injectHosts, keeping tools like gh and npm functional without exposing secrets.
  • MCP tool anthropic/requiresUserInteraction annotation — MCP server authors can mark individual tools as always requiring explicit human approval by setting _meta["anthropic/requiresUserInteraction"]: true in tools/list; Claude Code v2.1.199+ enforces this across all permission modes.
  • Web search & web fetch tool version bump to 20260318 — All code examples updated to reference web_search_20260318 and web_fetch_20260318; the new search version clarifies allowed_callers defaults, domain filtering mutual exclusivity, country code validation, and adds a request_too_large error code.
  • Subagents run in background by default (v2.1.198) — Claude Code now runs subagents in the background by default and surfaces permission prompts in the main session; the Explore built-in inherits the main conversation's model instead of always using Haiku.
  • Adaptive thinking display field added to examples — All SDK code samples for adaptive thinking now include "display": "summarized" (or equivalent) to make thinking content visible, reflecting that newer models default to "omitted".

Programmatic tool calling

Updated

Added a new "Input schema limitations" section documenting that tools whose input_schema contains a recursive $ref cannot be enabled for programmatic calling and will fail with a 400 Circular $ref detected error, with two documented workarounds.

  • Input schema limitations — New subsection documenting that recursive $ref cycles in a tool's input_schema are incompatible with programmatic calling via allowed_callers; the request fails with a 400 invalid_request_error containing Circular $ref detected.
  • Workarounds — Two options documented: omit allowed_callers to keep the tool direct-only, or remove the cycle by unrolling recursion to a fixed depth or replacing the recursive property with a plain {"type": "object"}.
