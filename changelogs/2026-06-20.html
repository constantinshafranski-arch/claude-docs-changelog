Connect Claude Code to tools via MCP

Updated

Replaces the environment-variable-only approach to disabling claude.ai connectors with a new `disableClaudeAiConnectors` settings key and clarifies its scope semantics, cloud session behavior, and per-connector blocking via `deniedMcpServers`.

  • disableClaudeAiConnectors setting — New JSON settings key disableClaudeAiConnectors: true (v2.1.182+) disables auto-fetching of claude.ai MCP connectors from any settings scope; uses any-source-true semantics so a project-level false cannot override a user- or policy-level true.
  • ENABLE_CLAUDEAI_MCP_SERVERS env var — The existing environment variable remains supported and now documented as a session-scoped alternative to the new settings key.
  • Per-connector blocking — Individual claude.ai connectors can be blocked by name or URL pattern via deniedMcpServers instead of disabling all connectors.
  • Cloud session scoping note — Added a note clarifying that disableClaudeAiConnectors does not apply to Claude Code on the web sessions, where connectors arrive as explicit --mcp-config entries and URL patterns are rewritten through the session proxy.

Model configuration

Updated

Documents new model retirement and remapping warnings emitted to stderr in non-interactive mode from v2.1.182, including suppression behavior for JSON output formats.

  • Retirement warnings in headless mode — From v2.1.182, model retirement or remapping warnings are written to stderr in non-interactive mode when using the default text output format; also covers models set in subagent frontmatter.
  • JSON output suppression — Retirement warnings are suppressed when --output-format json or stream-json is used; the actual model in use should be read from the modelUsage field of the result message instead.

Claude Code settings

Updated

Adds three new settings entries (`disableClaudeAiConnectors`, `remoteControlAtStartup`, `attribution.sessionUrl`), removes a startup-notice sentence about settings errors, and clarifies the attribution note to include `sessionUrl`.

  • disableClaudeAiConnectors — New setting (v2.1.182+) added to the available settings table; disables claude.ai MCP connector auto-fetch with any-source-true precedence rules.
  • remoteControlAtStartup — New setting (v2.1.119+) documented; controls whether Remote Control connects automatically at each interactive session start (true to always, false to never, unset to follow org default).
  • attribution.sessionUrl — New key added to the attribution settings table; controls whether the claude.ai session link is appended as a Claude-Session trailer on commits and as a link in PR descriptions; defaults to true.
  • Settings error notice — Removed the sentence stating that settings file errors trigger a setup issues notice at startup; only /status and /doctor references remain.
  • Attribution note — Updated the attribution precedence note to advise setting commit, pr, and sessionUrl all to empty/false to hide all attribution.
