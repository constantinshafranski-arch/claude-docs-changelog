Automate workflows with hooks

Updated

Documented hook execution semantics, interaction with permission modes, and limitations when multiple hooks modify tool inputs.

  • Multiple hooks execution behavior — Clarified that hooks run in parallel and most restrictive permission decision wins, with deny blocking even in bypassPermissions mode
  • Hook result merging rules — Documented that multiple PreToolUse hooks have non-deterministic ordering when rewriting tool arguments, and last one to finish wins
  • PreToolUse and permission mode interaction — Hooks fire before permission-mode checks and can enforce policy that users cannot bypass, but cannot loosen restrictions below permission rule limits
  • Command hook limitations expanded — Clarified that text from `additionalContext` is injected as system reminder that Claude reads as plain text, not as command/tool trigger
