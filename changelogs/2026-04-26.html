Use Claude Code Desktop

Updated

The Desktop overview was restructured around the three-tab app model (Chat, Cowork, Code), the session concept was clarified, Auto mode availability was moved to its own anchor section, SSH sessions now auto-install Claude Code remotely, and managed-settings scoping for Desktop vs. CLI/IDE was clarified.

  • App introduction — Opening paragraph rewritten to describe all three Claude Desktop tabs (Chat, Cowork, Code); the Code tab is now explicitly scoped as this page's subject.
  • Windows Git requirement — Git for Windows prerequisite surfaced in the post-install launch instructions rather than only in the session-isolation note.
  • Session concept — Desktop feature list replaced with a definition of a <em>session</em> as the core unit (own chat history, project folder, code changes) with a concise capability list.
  • Auto mode availability — Plan/model requirements moved out of the permissions table cell and into a dedicated auto-mode-availability anchor block for direct linking.
  • Preview pane media support — Video files added alongside HTML, PDF, and images as types the preview pane and file pane can open.
  • SSH auto-install — Desktop now installs Claude Code on the remote machine automatically on first connection; the requirement for manual pre-installation was removed.
  • Managed settings scope — Docs clarified that remotely pushed managed settings reach CLI and IDE sessions only; Desktop deployments must use MDM file distribution or admin console controls.
  • --allowedTools / --disallowedTools CLI comparison — Entry updated from "Not available in Desktop" to note that settings-file permission rules still apply even without a per-session equivalent.
  • Cowork tab reference — Git error troubleshooting note updated to link to the Cowork tab URL instead of referring to it as a generic session type.
  • Customize sidebar — Note added that connectors, skills, and plugins can all be managed from a single Customize button in the sidebar.
