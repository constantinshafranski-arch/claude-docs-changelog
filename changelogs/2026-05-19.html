Computer use tool

Updated

Added model-specific extended-thinking effort recommendations, a new click-miss diagnosis table, macOS Retina display guidance, a screenshot history management strategy for prompt caching, and a new tip to place instruction text before screenshots in the content array.

  • Extended thinking effort settings — New benchmarked guidance: use high effort for Claude Opus 4.7 (default) and medium for Claude Sonnet 4.6 and Opus 4.6; low on Sonnet/Opus 4.6 uses fewer output tokens than disabling thinking entirely.
  • Click-miss diagnosis table — New troubleshooting table maps symptoms (offset clicks, wrong element, poor accuracy) to likely causes and fixes including enable_zoom: true, display dimension mismatches, and DPI guidance.
  • macOS Retina display note — Added note that Retina screenshots capture at 2x device pixel ratio; downscale by 2x before sending or halve coordinates Claude returns before issuing the click.
  • Screenshot history and prompt caching — New section advises placing cache_control breakpoints after system prompt and on recent tool_result blocks, and pruning screenshots in batches (keep last 3, prune every 25 turns) to preserve cache prefix stability.
  • Content array ordering — New tip: place instruction text before the screenshot image in a user turn's content array to improve click accuracy.
  • Best practices card — Added link card to the Claude blog post with benchmarked recommendations for resolution, thinking effort, and context management.

Cache diagnostics

New

New beta feature (beta header: cache-diagnosis-2026-04-07) that diagnoses prompt cache misses by storing a lightweight fingerprint per response and comparing consecutive request structures; pass diagnostics.previous_message_id to identify exactly where the prompt prefix diverged across model, system prompt, tools, or message history.

  • diagnostics.previous_message_id — Pass the previous response id in this field to trigger a comparison; the response includes a diagnostics object with cache_miss_reason describing the first divergence point.
  • cache_miss_reason types — Response object reports divergence at one of: model, system prompt, tools, or message history.
  • Beta header required — Feature activated via anthropic-beta: cache-diagnosis-2026-04-07; available on Claude API only, not Bedrock or Vertex AI.
  • ZDR-qualified data retention — Fingerprints contain only hashes and token-count estimates (no raw prompt content), retained briefly and scoped to your organization and workspace.
  • Streaming support — Diagnostics work in both streaming and non-streaming modes.
  • SDK support — Code examples provided for Python, TypeScript, Go, C#, cURL, and CLI.
platform.claude.com/docs/en/build-with-claude/cache-diagnostics →
