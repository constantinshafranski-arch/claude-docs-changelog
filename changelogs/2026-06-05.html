Connect Claude Code to tools via MCP

Updated

Updated MCP documentation to fix sub-1000ms timeout behavior, clarify stdio option ordering, and document OAuth redirect limitations for Anthropic-hosted connectors.

  • Timeout floor removed — Per-server timeout values below 1000ms are no longer floored to 1 second; they now fall through to MCP_TOOL_TIMEOUT, defaulting to ~28 hours when unset (behavior change introduced in v2.1.162)
  • Option ordering clarification — Note rewritten to explain that -- separates Claude's own options from server arguments; warns that placing --env directly before the server name causes CLI to misread the name as a key=value pair
  • OAuth-only connectors — New note documents that Anthropic-hosted connectors such as Microsoft 365, Gmail, and Google Calendar cannot authenticate via local OAuth in Claude Code; from v2.1.162, the /mcp UI directs users to connect these at Settings → Connectors on claude.ai instead

Monitoring

Updated

Added tool_use_id and gen_ai.tool.call.id attributes to claude_code.tool and claude_code.tool.execution spans, and a skill.kind attribute to skill-invocation events.

  • tool_use_id attribute — New attribute on both claude_code.tool and claude_code.tool.execution spans carries the model's tool_use block ID, allowing spans to be joined with tool-result and tool-decision events and hook payloads
  • gen_ai.tool.call.id attribute — Added to both tool spans as an alias for tool_use_id conforming to the OpenTelemetry GenAI semantic convention
  • skill.kind attribute — New attribute on skill-invocation events; set to "workflow" for workflow skills and absent otherwise

Dreams

Updated

Added a dedicated 'Steer with instructions' section documenting how the optional instructions field influences the dreaming synthesis pipeline, and clarified its scope and limitations.

  • Steer with instructions section — New section explains that instructions is applied throughout the pipeline to guide what is read, merged, dropped, and how the output store is structured
  • High-level guidance only — Documents that instructions accepts focus areas, content-preservation rules, and output conventions, but that imperative line-level edits (e.g., "change sentence X to Y") generally have no effect; use the Memory Stores API for targeted edits
  • Capitalization fix — "Research Preview" normalized to "research preview" in the feature tip
