Connect Claude Code to tools via MCP

Updated

MCP roots/list now returns all additional working directories and Claude Code sends roots/list_changed notifications; the idle-abort timeout is extended to stdio servers as of v2.1.203.

  • roots/list expansion — roots/list now returns the session's launch directory plus every additional working directory added via --add-dir, /add-dir, or additionalDirectories; before v2.1.203 it returned only the launch directory.
  • notifications/roots/list_changed — Claude Code now sends notifications/roots/list_changed whenever the working-directory set changes; requires v2.1.203+.
  • CLAUDE_PROJECT_DIR stability — Clarified that CLAUDE_PROJECT_DIR is the stable project root and does not change when directories are added or removed mid-session.
  • MCP idle timeout for stdio servers — The idle-abort timeout (default 30 min for stdio, 5 min for HTTP/SSE/WS) now applies to all server types except IDE and SDK in-process servers; stdio servers were exempt before v2.1.203.
  • per-server timeout as idle floor — A per-server timeout of at least 1000 ms now also acts as a floor on the idle timeout, preventing idle aborts sooner than the configured wall-clock limit; requires v2.1.203+.

Claude Code on Microsoft Foundry

Updated

A third authentication method (Option C) is added for Microsoft Foundry: bearer token authentication via ANTHROPIC_FOUNDRY_AUTH_TOKEN, which takes precedence over the API key and default credential chain (v2.1.203+).

  • ANTHROPIC_FOUNDRY_AUTH_TOKEN — New Option C bearer token auth; Claude Code sends this value as Authorization: Bearer on every request, useful when a host process has already obtained an Entra ID token; requires v2.1.203+.
  • authentication method count — Documentation updated from two to three supported authentication methods.
  • Entra ID credential chain fallback — Condition for falling back to the Azure SDK default credential chain updated to require that neither ANTHROPIC_FOUNDRY_API_KEY nor ANTHROPIC_FOUNDRY_AUTH_TOKEN is set.
  • precedence order — ANTHROPIC_FOUNDRY_AUTH_TOKEN takes precedence over both ANTHROPIC_FOUNDRY_API_KEY and the default credential chain.

Claude Code settings

Updated

Multiple settings entries are updated for v2.1.203: cleanupPeriodDays gains a parse-error safety guard, ultracode description is clarified, worktree.bgIsolation documents non-git hook failure behavior, and workflowSizeGuideline now controls the Large workflow warning threshold.

  • cleanupPeriodDays parse-error guard — If Claude Code cannot read or parse a settings file it now pauses the retention cleanup sweep and shows a warning in /status; prevents transcript deletion when a broken settings file would otherwise drop to the 30-day default.
  • ultracode setting — Description updated to note that claude --effort ultracode starts a session with ultracode already on; requires v2.1.203+.
  • worktree.bgIsolation non-git fallback — Documented that a failing WorktreeCreate hook in a non-git repo releases the file-edit block so the session can edit the working directory in place; requires v2.1.203+.
  • workflowSizeGuideline large-workflow threshold — The guideline's agent count now also replaces the default threshold for the Large workflow cost warning; requires v2.1.203+.
