User Profiles

New

New Java SDK beta reference for the /v1/user_profiles endpoint, covering create, list, retrieve, update, and enrollment URL operations with full parameter and return-type documentation for the BetaUserProfile resource.

  • POST /v1/user_profiles — beta().userProfiles().create() accepts optional externalId (max 255 chars) and metadata (max 16 key-value pairs) and returns a BetaUserProfile with uprof_-prefixed id, RFC 3339 timestamps, and trustGrants map.
  • GET /v1/user_profiles — beta().userProfiles().list() supports limit, order (asc/desc), and page pagination parameters and returns a UserProfileListPage.
  • GET /v1/user_profiles/{user_profile_id} — beta().userProfiles().retrieve() fetches a single profile by ID.
  • trustGrants — Per-profile trust grant map with statuses ACTIVE, PENDING, or REJECTED; key omitted when no grant is active or in flight.
  • Beta header — All operations require the user-profiles-2026-03-24 beta flag via AnthropicBeta.USER_PROFILES_2026_03_24.
