Key Highlights

  • Cloud session traceability — Claude Code web sessions now expose CLAUDE_CODE_REMOTE_SESSION_ID, enabling traceable links to session transcripts in PR bodies, commit messages, and reports.
  • InstallConfigurator overhaul — The Quickstart and Overview install widget gained a defaultSurface prop, per-target taglines, a redesigned shell switcher (PowerShell/CMD toggle), and polished handoff card styling.
  • Cost estimate clarity — The /cost command and cost.total_cost_usd statusline field are now explicitly documented as client-side estimates that may differ from actual billing; the Claude Console Usage page is cited for authoritative figures.
  • Sandbox network settings clarified — network.allowUnixSockets is now marked macOS-only; Linux/WSL2 users must use allowAllUnixSockets (which bypasses the seccomp filter blocking AF_UNIX sockets).
  • Extended thinking progress hints — During extended thinking, progress hints now appear below the indicator so users can see that Claude is actively working.
  • JetBrains diff tool options — The /config diff tool setting now documents both auto (show diffs in IDE) and terminal (keep diffs in terminal) as explicit choices.

Claude Code settings

Updated

Clarified platform-specific behavior of sandbox Unix socket settings, marking network.allowUnixSockets as macOS-only and explaining why Linux and WSL2 must use allowAllUnixSockets instead.

  • network.allowUnixSockets — Now documented as macOS-only; on Linux and WSL2 the seccomp filter cannot inspect socket paths, so this setting is ignored.
  • network.allowAllUnixSockets — Expanded description clarifies that on Linux and WSL2 this is the only way to permit Unix sockets, as it skips the seccomp filter that otherwise blocks socket(AF_UNIX, ...) calls.
