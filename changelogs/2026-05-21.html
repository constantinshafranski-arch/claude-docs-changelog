Hooks reference

Updated

Overhauled the hooks reference with corrected exit-0 semantics, new SessionStart output fields, suppressOriginalPrompt for UserPromptSubmit, expanded decision-pattern table, and promotion of PermissionDenied and TeammateIdle to full five-hook-type support.

  • Exit 0 semantics — Consistently updated all inline comments and prose from "allow the tool call" to "no decision; normal permission flow applies"
  • suppressOutput clarification — Field now documented as hiding stdout from the transcript only; stdout still appears in the debug log
  • SessionStart new output fields — Added initialUserMessage (sets the first user turn in non-interactive mode) and watchPaths (array of paths to watch for FileChanged events) to the SessionStart hook output table
  • suppressOriginalPrompt — New UserPromptSubmit output field; when true with decision: "block", omits the original prompt text from the block message shown to the user
  • Decision-pattern table — Added new row for SessionStart, Setup, and SubagentStart events using context-only pattern via hookSpecificOutput.additionalContext
  • setMode values — Added auto as a valid permission mode in the PermissionRequest setMode operation
  • Event hook-type promotion — PermissionDenied and TeammateIdle moved from command/http/mcp_tool-only list to all-five-types list (adding prompt and agent hook support)
  • TeammateIdle block behavior — Added documentation: by default blocks stop the teammate; set continueOnBlock: true to feed the reason back instead
  • PermissionDenied output — Documented that ok: false has no effect; only command hooks can set hookSpecificOutput.retry; prompt and agent hook output is discarded on this event
  • session_crons source — Added ScheduleWakeup as a source for session-scoped cron entries alongside CronCreate and /loop
