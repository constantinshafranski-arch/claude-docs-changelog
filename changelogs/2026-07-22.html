  • New doc: Get your Claude API key — Dedicated guide covering API key creation, workspace scoping, expiration, environment variable setup, and the distinction between Claude API keys and Admin API keys.

Hooks reference

Updated

  • PreToolUse block behavior — ok: false now ends the turn by default; set continueOnBlock: true to return the deny reason to Claude as a tool error (equivalent to permissionDecision: "deny"). Before v2.1.210, the deny reason was always returned to Claude.

Monitoring

Updated

Added new OTel environment variable CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH, new event attributes message.uuid and client_request_id, fixed metric unit labels for Prometheus, and improved OTel setup verification guidance.

  • Setup verification guidance — Added instructions to check for claude_code.session.count metric (metrics setup) or claude_code.user_prompt event (logs-only setup) to verify OTel export.
  • CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH — New env var (v2.1.214+) sets the maximum UTF-16 code unit length for content-bearing attributes; defaults to 61440 (60 KB), interacts with OTEL_ATTRIBUTE_VALUE_LENGTH_LIMIT.
  • message.uuid attribute — New attribute (v2.1.214+) on user_prompt and assistant_response events carries the UUID of the persisted session transcript entry.

  • cost_usd_micros on API request events — New integer attribute emitting estimated cost in millionths of a USD.
  • Prometheus unit fix — As of v2.1.216, when Prometheus is the sole exporter, Claude Code omits USD, tokens, and s unit lines to stay valid Prometheus text format; before v2.1.216 some scrapers rejected the scrape.
  • Metric unit column corrected — Count metrics (session.count, lines_of_code.count, pull_request.count, commit.count, code_edit_tool.decision) now show none instead of count in the unit column.
  • Example configurations restructured — Each example scenario is now a separate labeled code block instead of a single commented block.

  • sandbox.filesystem.disabled — New boolean setting (v2.1.216+) disables filesystem isolation while preserving network isolation; honored only from user, managed, or CLI settings—not project settings.
