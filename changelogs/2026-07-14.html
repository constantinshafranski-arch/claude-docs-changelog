Key Highlights

  • Compliance API activity types — 18 new activity types added across agent proxy provisioning lifecycle, Claude chat sync sources, Cowork auto mode, workspace inference data retention, Slack workspace claims, and plugin enable/disable events, bringing the total filter set to 399+.
  • Artifact read_mode — A new "public" value is available in the read_mode field across the Code Artifacts API, allowing artifacts to be visible to anyone on the internet.
  • Amazon Bedrock / Google Cloud credential and model defaults (v2.1.207) — Default primary model upgraded to Opus 4.8 on all cloud providers; background tasks now use a dedicated Sonnet small/fast model; AWS credential chain is cached in memory with a 60-second timeout and CLAUDE_CODE_SKIP_AWS_CRED_CACHE override.
  • HIPAA self-serve enablement — Eligible organizations can now review, execute, and immediately apply Anthropic's standard BAA directly from Claude Console → Settings → Privacy, removing the need to contact sales for standard HIPAA-ready API access.
  • Claude Code settings security hardening (v2.1.207) — autoMode, pluginConfigs, and plugin ${user_config.*} substitutions are now read only from user/managed settings; project and local settings are ignored to prevent cloned repositories from supplying these values.
  • MCP and plugin hardening — Shell-form plugin hook commands and headersHelper can no longer reference ${user_config.*} values; untracked .claude/settings.local.json MCP approvals now require the workspace trust dialog before taking effect.

Claude Code on Amazon Bedrock

Updated

Major updates for v2.1.207: default primary model upgraded from Sonnet 4.5 to Opus 4.8, a dedicated Sonnet 4.5 small/fast model replaces the previous "same as primary" fallback, AWS credential chain is now cached with a 60-second timeout, and the startup model fallback now cascades to Sonnet when no Opus variant is available.

  • Default model table (v2.1.207) — Primary model remains Opus 4.8; small/fast model is now us.anthropic.claude-sonnet-4-5-20250929-v1:0 instead of the primary model.
  • Credential caching (v2.1.207) — AWS default credential chain is resolved once and cached in memory until five minutes before expiry (or one hour when no expiry is set); a credential error clears the cache. CLAUDE_CODE_SKIP_AWS_CRED_CACHE=1 restores per-request resolution.
  • Credential resolve timeout — Each chain resolution times out after 60 seconds; CLAUDE_CODE_AWS_CHAIN_RESOLVE_TIMEOUT_MS raises the limit for interactive SSO flows.
  • awsCredentialExport behavior (v2.1.206) — When set without awsAuthRefresh, startup no longer re-resolves the default provider chain, preventing multi-minute startup delays on restricted egress networks.
  • Model fallback improvement — Startup fallback now tries earlier Opus versions first and falls back to the default Sonnet model when no Opus version is accessible.
  • Background task model selection — Two rules govern which model runs background tasks; setting ANTHROPIC_DEFAULT_OPUS_MODEL without ANTHROPIC_DEFAULT_SONNET_MODEL now also redirects background tasks to the selected model.
  • Billing warning — New warning: deployments not pinning a primary model will be billed at Opus rates after v2.1.207.
  • 1M context window link — Updated anchor to #context-window-sizes-by-model.
  • Mantle version note removed — Prerequisite note requiring v2.1.94 removed.

Updated

Multiple security and behavioral fixes: untracked .claude/settings.local.json MCP approvals now require workspace trust; OAuth servers retry once on 401 before re-prompting; missing env var substitutions now warn instead of failing the config; and plugin headersHelper can no longer use ${user_config.*}.
