Key Highlights

  • Compliance API activity types expanded — 42 new activity types added across Claude Code agent lifecycle events (ccr_agent_*), CMEK key management (org_external_key_*), trusted device enrollment, step-up authentication, plugin directory submissions, and more; activity_types filter now covers 384 values.
  • Code Artifacts API path migration — All Code Artifact endpoints moved from /v1/compliance/code/artifacts to /v1/compliance/apps/code/artifacts; organization_uuid query param removed from download and delete routes; organization_id field removed from list response in favor of organization_uuid.
  • Chat list endpoint gains order_by parameter — GET /v1/compliance/apps/chats now accepts order_by=created_at|updated_at for org-wide queries, enabling incremental exports sorted by last-modified time without enumerating users first; backward pagination restricted to per-user queries.
  • Compliance API key creation extended to organization owners — Claude Enterprise organization owners (not just primary owners) can now create Compliance Access Keys scoped to their own organization; "Get access to the Compliance API" page renamed to "Set up the Compliance API" with a revised step-by-step setup flow.
  • Organization settings response adds new boolean and string setting types — 12 new boolean settings added (e.g., claude_code_web_enabled, claude_code_workflows_enabled, claude_design_enabled); new String setting type introduced for claude_code_default_worker_environment_id and claude_code_default_worker_pool_id; expires_at field added to compliance API key objects.
  • Agent Skills and MCP tunnel artifacts updated — Skills API now uses code_execution_20260521 tool type (GA, no beta header required); MCP proxy image digest bumped and Helm chart updated to version 2.0.1; Java SDK examples added to advisor tool docs.
