Key Highlights

  • WebSearch tool unavailable on Bedrock — Claude Code on Amazon Bedrock now explicitly documents that the WebSearch tool is not available, with a link to the WebSearch tool behavior reference.
  • overloaded error category added — The system/api_retry stream event, StopFailure hook matcher, and hooks reference tables all gain overloaded as a recognized error type alongside rate_limit, enabling hooks and CI scripts to differentiate server-overload failures.
  • OTEL_METRICS_INCLUDE_RESOURCE_ATTRIBUTES — New env var lets operators exclude OTEL_RESOURCE_ATTRIBUTES keys from metric datapoint labels to control cardinality, while custom attributes are still sent in the OTLP resource block.
  • forceLoginMethod / forceLoginOrgUUID updated — Both managed settings now explicitly exempt Bedrock, Vertex, and Foundry third-party provider sessions from first-party OAuth enforcement, clarifying that cloud IAM handles those cases.
  • /fork command now on by default (v2.1.161+) — Forked subagents are no longer experimental; CLAUDE_CODE_FORK_SUBAGENT is only required to make forks the model's default spawn behavior, and /fork works without it.
  • Vision image size limits clarified — The Claude API per-image limit is raised to 10 MB (previously documented as 5 MB); Amazon Bedrock and Vertex AI remain at 5 MB, with new inline documentation and an updated FAQ entry.

Build an orchestration mode

Updated

The orchestration mode example was significantly expanded across all SDK languages to add a PHP port, a JOURNAL_PATH for task tracking, separate MAX_CONCURRENT and MAX_TOTAL_SUBTASKS constants, a REQUEST_TIMEOUT_SECONDS setting, a DOC_TEST_MODE flag for CI validation, and atomic journal writes.

  • MAX_CONCURRENT / MAX_TOTAL_SUBTASKS split — Replaced the single MAX_PARALLEL_AGENTS constant with separate MAX_CONCURRENT (concurrency cap) and MAX_TOTAL_SUBTASKS (planning budget) constants across Python, TypeScript, C#, Go, Java, and Ruby.
  • DOC_TEST_MODE flag — Added a DOC_TEST_MODE environment variable check across all ports that caps loops to a single turn, allowing the automated docs harness to validate examples without running the full orchestration.
  • JOURNAL_PATH / orchestration journal — Added JOURNAL_PATH (defaulting to orchestration_journal.json, overridable via ORCH_JOURNAL) and atomic journal write logic across all SDK ports.
  • REQUEST_TIMEOUT_SECONDS — Added a 600-second request timeout constant across all ports, with the C# port also setting a non-streaming requestMaxTokens of 8192 to stay within API limits.
  • PHP port added — A new PHP implementation of the orchestration example is included in the CodeGroup, covering all the same constants and patterns as the other language ports.
