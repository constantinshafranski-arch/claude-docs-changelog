Key Highlights

  • managed-agents-2026-04-01 beta flag — A new beta version string has been added to the AnthropicBeta enum across all API endpoints (REST, CLI, Python, TypeScript, Go, Java, Ruby, C#), enabling the managed agents feature set.
  • multiagent coordinator topology — The Agents API now supports a multiagent body parameter on create/update, enabling a coordinator pattern where a primary session thread can spawn up to 20 sub-agent threads from a defined roster, with support for recursive self-invocation via {"type":"self"}.
  • Session Threads API (new, 7 SDKs) — New /v1/sessions/{session_id}/threads endpoints (archive, list, retrieve, events/list, events/stream) have been added across all SDK flavors, allowing management of named sub-threads within a session.
  • Webhooks API (new, all SDKs) — New /v1/webhooks and SDK-specific webhook unwrap helpers have been published for Python, TypeScript, Go, Ruby, enabling event-driven integrations.
  • MCP OAuth validate endpoint (new, all SDKs) — A new POST /v1/vaults/credentials/mcp_oauth_validate endpoint has been added across all SDK flavors for validating MCP OAuth credentials.
  • Document citation block fields enriched — The cited_text, start_block_index, and end_block_index fields on citation objects now include explicit documentation on their semantics (exclusive end index, concatenated text, token counting rules) in the Messages and Message Batches API docs.
