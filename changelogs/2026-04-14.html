Key Highlights

  • PreCompact hook blocking — PreCompact now supports exit code 2 and decision: "block" to block compaction; moved from the no-decision group to the top-level decision pattern group alongside UserPromptSubmit, Stop, and others
  • asyncRewake hook field — New asyncRewake field on command hooks runs the hook in the background and wakes Claude immediately when the hook exits with code 2, enabling reactive long-running background failure handling
  • Subagent status lines — New subagentStatusLine setting renders a custom row body for each subagent in the agent panel, accepting a command that receives task data via stdin and returns per-row JSON overrides
  • Vertex AI EU multi-region endpoint — eu is now a supported multi-region identifier alongside us, routing to https://aiplatform.eu.rep.googleapis.com for EU data-residency deployments
  • Plugin monitors support — Plugins can now declare a monitors field in plugin.json pointing to background Monitor configurations that auto-arm at session start or on skill invocation; default path is monitors/monitors.json
  • Skill when_to_use field &amp; enlarged description cap — New optional when_to_use frontmatter field appends additional invocation context to skill listings; combined description cap raised from 250 to 1,536 characters

Hooks reference

Updated

Significant updates to hook blocking semantics, the PreCompact event, asyncRewake for background hooks, SessionEnd timeout configuration, and PreToolUse permission evaluation clarifications.

  • PreCompact blocking — PreCompact now supports exit code 2 to block compaction and the top-level decision: "block" JSON pattern; moved from the no-decision group to the decision-capable group in the reference table
  • PreCompact behavior detail — Added documentation explaining that blocking proactive compaction skips it silently, while blocking recovery compaction surfaces the underlying API context-limit error
  • asyncRewake field — New asyncRewake: true field on command hooks runs in the background and wakes Claude immediately on exit code 2, with stderr shown to Claude as a system reminder
  • PreToolUse permissionDecision clarification — Updated description of permissionDecision to clarify that deny and ask permission rules are still evaluated regardless of what the hook returns
  • SessionEnd timeout — Replaced the global CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS-only approach with per-hook timeout settings; overall budget is now auto-raised to the highest per-hook timeout (up to 60 s), with plugin-provided hooks excluded from raising the budget
