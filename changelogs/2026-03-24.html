Sandboxing

Updated

Comprehensive documentation on Claude Code's OS-level sandbox for Bash commands.

  • Filesystem isolation — restricts writes to CWD by default; configurable via sandbox.filesystem.allowWrite/denyWrite/denyRead/allowRead
  • Network isolation — proxy-based domain filtering; new domain requests trigger permission prompts
  • OS-level enforcement — Seatbelt on macOS, bubblewrap on Linux/WSL2
  • Two modes — Auto-allow (sandboxed commands run without prompts) and Regular permissions
  • Security benefits — protects against prompt injection, malicious dependencies, social engineering
  • Escape hatch — dangerouslyDisableSandbox for incompatible tools; disableable with allowUnsandboxedCommands: false
  • Open source — sandbox runtime available as @anthropic-ai/sandbox-runtime npm package

Tool Search

New

Scale agents to up to 10,000 tools with on-demand discovery instead of loading everything upfront.

  • How it works — tool definitions withheld from context; agent searches and loads 3–5 relevant tools per query
  • Enabled by default — configure via ENABLE_TOOL_SEARCH env var (true / auto / auto:N / false)
  • Auto mode — activates when tool definitions exceed 10% of context window; auto:N for custom threshold
  • Optimization — use descriptive tool names and keyword-rich descriptions for better discovery
  • Limits — max 10,000 tools, 3–5 results per search, requires Sonnet 4+ or Opus 4+ (no Haiku)

Context Editing

Updated Beta

New beta feature for fine-grained server-side conversation context management.

  • Tool result clearing (clear_tool_uses_20250919) — automatically clears oldest tool results when context exceeds threshold; ideal for agentic workflows with heavy tool use
  • Thinking block clearing (clear_thinking_20251015) — manage extended thinking blocks with configurable preservation (keep: "all" for cache hits, or {type: "thinking_turns", value: N})
  • Server-side processing — editing happens before the prompt reaches Claude; client maintains full unmodified history
  • Client-side SDK compaction — alternative summary-based approach via tool_runner in Python/TS/Ruby SDKs
  • Beta header required — context-management-2025-06-27
  • Model support — all Claude 4.x models (Opus 4–4.6, Sonnet 4–4.6, Haiku 4.5)
