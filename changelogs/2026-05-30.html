Key Highlights

  • Cloud sandbox reference (new doc) — New reference page documents pre-installed languages (Python 3.12+, Node 20+, Go, Rust, Java, Ruby, PHP, C/C++), databases, utilities, and sandbox specs (Ubuntu 22.04, 8 GB RAM, 10 GB disk) for Managed Agents cloud environments.
  • MCP WebSocket transport — Claude Code now supports type: "ws" MCP servers via .mcp.json or claude mcp add-json, enabling persistent bidirectional connections to event-pushing remote servers.
  • Skills-directory plugins + plugin init — Any skill folder containing a .claude-plugin/plugin.json manifest auto-loads as a &lt;name&gt;@skills-dir plugin; claude plugin init scaffolds the structure with optional component flags (--with skills hooks mcp etc.).
  • Multiagent coordinator pinning clarification — Coordinator multiagent.agents rosters are snapshotted at creation time; updating a referenced agent does not propagate automatically—you must update the coordinator to pick up new versions.
  • Memory store 2,000-memory limit + best practices — Memory stores now enforce a hard 2,000-memory cap with documented recovery strategies (condense with dreaming sessions, attach fresh stores, use read-only access for reference material).
  • Claude Platform on AWS expansion — Webhooks, multiagent orchestration, and self-hosted sandboxes are now available on Claude Platform on AWS; a new AnthropicSelfHostedEnvironmentAccess managed IAM policy is added for worker authentication.
