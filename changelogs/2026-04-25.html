Release Notes

Key Highlights

  • Rate Limits API (new) — New Admin API endpoints at GET /v1/organizations/rate_limits and GET /v1/organizations/workspaces/{id}/rate_limits allow programmatic querying of organization- and workspace-level rate limits, with filtering by group_type or specific model alias.
  • API key expiry — APIKey objects now include an expires_at RFC 3339 field and an "expired" status value, surfaced across Retrieve, List, and Update endpoints.
  • Role cleanup — The "managed" role variant has been removed from invite, user, and update endpoints; the explicit enum now resolves to "claude_code_user" as the fourth assignable role.
  • Usage Report enhancements — GET /v1/organizations/usage_report/messages gains account_ids and service_account_ids filter params, plus "account_id" and "service_account_id" group-by dimensions; the speed field (fast-mode research preview) is removed.
  • Claude on Amazon Bedrock (legacy) — The legacy Bedrock integration (InvokeModel / Converse APIs with ARN-versioned model IDs) has been split into its own dedicated page at claude-on-amazon-bedrock-legacy, with all SDK cross-links updated accordingly.
  • Claude Code settings — Five settings previously stored only in ~/.claude.json (autoScrollEnabled, editorMode, showTurnDuration, teammateMode, terminalProgressBarEnabled) are now first-class entries in settings.json; new prUrlTemplate, autoScrollEnabled, and editorMode keys documented.
