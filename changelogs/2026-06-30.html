Key Highlights

  • Microsoft Foundry hosting options — Across 10+ docs, advanced features (Agent Skills, MCP connector, code execution, web fetch/search, Files API, programmatic tool calling, structured outputs) now require a "Hosted on Anthropic" deployment on Microsoft Foundry; the "beta" label has been removed, reflecting GA status.
  • Claude apps gateway — New self-hosted gateway is referenced throughout Claude Code CLI docs, adding SSO/IdP sign-in, per-user spend limits, OTLP telemetry with user attribution, and managed settings delivery in front of Bedrock, Vertex AI, Foundry, or the Anthropic API.
  • Fine-grained tool streaming — Major doc rewrite with per-tool `eager_input_streaming` field replacing the legacy beta header, expanded code examples across Python, TypeScript, C#, and Go that now print streamed fragments, and clarified behavior for partial/invalid JSON and `max_tokens` truncation.
  • Tool search tool — Added explicit model compatibility table (Fable 5 through Haiku 4.5), rewrote introduction with quantified context-bloat and accuracy figures, and clarified `defer_loading` semantics and quick-start walkthrough.
  • Claude Code hooks (v2.1.195/v2.1.196) — Hooks reference updated with hyphen support in exact-match matchers, new `prompt_id` common field for OTel correlation, `MessageDisplay` event added to no-matcher list, and plugin-scoped agent type naming for `SubagentStart`/`SubagentStop`.
  • Claude Code settings — Multiple new settings added: `autoMode.classifyAllShell`, `disableSideloadFlags`, `enableArtifact`, `forceLoginGatewayUrl`, `skillListingMaxDescChars` (renamed from `maxSkillDescriptionChars`), and `forceLoginMethod` now accepts `gateway`.

Using agent memory

Updated

Clarifies how memory store mount paths are derived from the store's display name and documents that writes outside the mount path land in ephemeral container scratch space.
