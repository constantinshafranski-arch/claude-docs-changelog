CLI reference

Updated

Added two new MCP OAuth commands (claude mcp login and claude mcp logout) and a new iterm2 option for the --teammate-mode flag, both requiring Claude Code v2.1.186.

  • claude mcp login &lt;name&gt; — New command (v2.1.186+) that runs a configured MCP server's OAuth flow from the shell; supports --no-browser for SSH environments to print the authorization URL instead of opening a browser.
  • claude mcp logout &lt;name&gt; — New command (v2.1.186+) that clears stored OAuth credentials for a named MCP server.
  • --teammate-mode iterm2 — Added iterm2 as a new display mode value (v2.1.186+) that uses iTerm2 native split panes via the it2 CLI.

Create custom subagents

Updated

Revised background subagent permission handling to reflect v2.1.186 behavior where permission prompts now surface in the main session, and updated SendMessage tool availability semantics.

  • Background subagent permission prompts — As of v2.1.186, when a background subagent hits a tool call requiring permission, the prompt surfaces in the main session and names the subagent; press Esc to deny that one call without stopping the subagent. Prior to v2.1.186, all such calls were auto-denied.
  • CLAUDE_CODE_FORK_SUBAGENT behavior — Updated to note that permission prompts from fork-mode background subagents also surface in the main session, consistent with the v2.1.186 change.
  • SendMessage availability — Clarified that SendMessage is always available for resuming subagents by agent ID or name; structured team-protocol messages (e.g., shutdown_request) still require agent teams to be enabled.
  • Fork vs. named subagent table — Updated the Permissions row to reflect that named subagents running in the background now surface prompts in the main session rather than auto-denying.
