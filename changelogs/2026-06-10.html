Key Highlights

  • Deployments API (new) — Full CRUD for scheduled/managed agent deployments added across all SDKs (Python, TypeScript, Ruby, Go, Java, C#, PHP, CLI), including create, list, get, update, archive, run-now, pause, and unpause endpoints at POST /v1/deployments.
  • Deployment Runs API (new) — New GET /v1/deployment_runs and GET /v1/deployment_runs/{id} endpoints allow listing and retrieving individual deployment execution records, with filtering by deployment_id and has_error.
  • BetaManagedAgentsModel — claude-mythos-5 removed from the model enum across all agent endpoints; claude-opus-4-7 added and the claude-fable-5 description updated to "Next generation of intelligence for the hardest knowledge work and coding problems".
  • New beta flags — Two new anthropic-beta header values added to all beta endpoints: server-side-fallback-2026-06-01 and fallback-credit-2026-06-01, raising the total count from 23 to 25.
  • Environments API — All environment CRUD and work-queue endpoints updated with the two new beta flags; no behaviour changes otherwise.
  • Volume note — This release includes 322 docs touched (286 updated + 36 new); per-SDK sub-entries for Deployments and Deployment Runs across CLI, C#, Go, Java, PHP, Python, Ruby, and TypeScript were omitted from the entry list due to volume but are fully published.

Deployments

New

New top-level reference page covering the full Deployments resource: create, list, get, update, archive, run-now, pause, and unpause endpoints for managing scheduled agent deployments.

  • POST /v1/deployments — Create a new deployment with agent, schedule, system prompt, and context configuration.
  • GET /v1/deployments — List deployments with filters for agent_id, status (active/paused), date range, and include_archived.
  • GET /v1/deployments/{id} — Retrieve a single deployment by ID.
  • POST /v1/deployments/{id} — Update mutable deployment fields; omit fields to preserve current values.
  • POST /v1/deployments/{id}/archive — Archive a deployment, preventing future scheduled runs.
  • POST /v1/deployments/{id}/run — Trigger an immediate ad-hoc run of the deployment, returning a DeploymentRun object.
  • POST /v1/deployments/{id}/pause — Pause a deployment so scheduled runs are skipped.
  • POST /v1/deployments/{id}/unpause — Resume a previously paused deployment.
platform.claude.com/docs/en/api/beta/deployments →
